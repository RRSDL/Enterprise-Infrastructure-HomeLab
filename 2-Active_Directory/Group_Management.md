# Security Groups

## Objective

Implement Role-Based Access Control (RBAC).

## Groups Created

- HR-Users
- IT-Users
- Finance-Users

## Membership

HR Users
→ HR-Users

IT Users
→ IT-Users

Finance Users
→ Finance-Users

## Why

Permissions should be assigned to groups rather than individual users.

Model:

User
↓
Group
↓
Resource

Benefits:

- Easier administration
- Better scalability
- Industry standard approach

## Result

Department-based access control implemented.
